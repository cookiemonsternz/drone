### VTX LDO

I'm going to go with another BL8072 chip for the VTX LDO, they're very easily available on JLC and LCSC, and they're super super easy to set up (no external feedback network) which is a big plus. The exact one is the BL8072CLTR33 - Just the 3.3V version of the chip.

### IMU LDO

This one has some specific reccommended requirements from the betaflight docs, so I'll have a look and see if the BL8072 meets them and if not I'll try and find a chip that does.

Unfortunately, the chip doesn't specify the noise characteristics in the datasheet, and this is rather important for the IMU, so I'm going to find another one which does.

The [MD7601A50](https://jlcpcb.com/partdetail/3391899-MD7601A50/C2978483) is looking reasonably promising; It's got a max current of 1A, and a psrr of ~80dB at 1Hz, which is well above the reccomended spec. It is worth noting that the psrr was measured with a Vout of 3.6V whereas mine would be 5V, but hopefully this doesn't cause a 10dB difference (which is like twice as loud I think?). Anyways, I'm going to go with this one.

### IMU Interrupt Wiring

I finally got around to looking in the hardware development channel on the discord and my goodness there is a lot of good material :) I someewhat quickly figured out what to do with the IMU interrupts. One pin (INT2) is actually the clock input for the IMU, which I've connected to TIM10, a random generic timer on the MCU. It should work but I'll have a look round for any other target files doing the same thing, pretty easy to move it later if necessary. The other pin is just an interrupt which goes to any gpio on the MCU, defined in the target file. For now I'm just copying the target file for the [Acci FPV](https://www.pcbway.com/project/shareproject/Acci_FPV_Flight_Controller_Betaflight_STM32F405_AT32F435_48fb0ecb.html) FC, bc its pretty overwhelming to just start chucking stuff in with no rhyme or reason lol.

_I'm considering moving the time to TIM1_CH1N if its compatible, because it is physically next to the IMU int pin, but I could just as easily move the IMU int pin._

### OSD Clock

Did some more hunting around and found that I probably should be using an external clock for the OSD chip; Also it seems that HSYNC and VSYNC aren't actually required, and the chip can determine it from VIN (I keep on confusing this for voltage lol, its video).

The required clock is 27MHz which should be easy enough, just some additional considerations with placement but I do tend to find them reasonably forgiving (albeit I've only used them in low frequency applications, with the rf stuff nearby it might be a bit more important, although that is 5.8GHz compared to 27MHz, could be less interplay?)

### ELRS 
ELRS just connects over UART and is typically set up as a generic UART port on the flight controller which the user can plug any uart peripheral into and then configure in betaflight. This poses an interesting consideration which is whether I want these generic peripheral ports, as this is a tinywhoop controller and my board space is very limited, I have a feeling its going to be a matter of if I've got space then sure.

For now I'll continue as if I'm only having ports for predetermined peripherals.

Anyways, I've connected the ELRS port to UART1.
